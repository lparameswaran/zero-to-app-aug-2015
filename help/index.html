HTML structure

HTML is used to define the skeleton of your app. Browser renders HTML files. A sample HTML document format is as follows: 
      <!DOCTYPE html>
      <html>
        <head>
          <title>First Page</title>
        </head>
        <body>
          <h1>Bold heading in my first page</h1>
          <h6>Bold, but smaller text in my first page</h1>
        </body>
      </html>
This document will be rendered in the browser as follows:

Bold heading in my first page

Bold, but smaller text in my first page

HTML tags

HTML is used to define the skeleton of your app. Browser renders HTML files. A HTML document uses tags - each tag has a beginning (ex., <html>) and an end (ex., </html>). A HTML document may contain many different tags, some of the most important ones are shown below:
  1. !DOCTYPE html tag at the top (identifies a HTML5 document).
  2. A html tag that identifies the start and end of the document.
  3. A head tag that identifies the heading section of the document.
    • Within this tag, usually title tag is used by the browser to identify the tab.
    • Javascript code/files and css code/files can also be added.
  4. A body tag that identifies the main (body) section of the document. This body will contain portions of the document that will be rendered in the browser.

    Within the body tag, you may see the following tags:
    • h1, h2 ... h6 tags that identify section headings from largest to smallest.
    • b, u and i tags for making text bold, underline and italics respectively.
    • p and br tags for marking a paragraph and to give a line break (new line) respectively.
    • Lists:
      • Ordered (numbered) lists: Start and end with a ol tag. Each item in the list should start and end with li tag.
      • UnOrdered (bulleted) lists: Start and end with a ul tag. Each item in the list should start and end with li tag.
    • Tables start and end with a table tag. Within the table, you may have:
      • Optional thead tag for the heading row(s). Each heading row has a starting and ending tr tag. Inside the heading row(s), you may have multiple th tags, one for each column.
      • tbody tag for the body row(s). Each body row has a starting and ending tr tag. Inside the body row(s), you may have multiple td tags, one for each column.
      • Optional tfoot tag for the footer row(s). Each footer row has a starting and ending tr tag. Inside the footer row(s), you may have multiple td tags, one for each column.
    • b, u and i tags for making text bold, underline and italics respectively.

CSS

Cascading Style Sheets define the beauty (skin) of your app. CSS refers to HTML elements either by their tag name (not usual) OR by id (#) or by class (.). A typical CSS may control a tag's class as follows:
In HTML: <h1 class="xyz"> .... </h1>
In CSS: .xyz { color: red, background-color: blue; } which will set that particular h1 tag to be rendered with red on blue color.

A typical CSS may control a tag identified by id abc as follows:
In HTML: <div id="abc"> .... </div>
In CSS: #abc { color: red; font-size: 200%; } which will set that particular div tag to be rendered with red color and double the usual font size.

Javascript

Javascript is like the internal organs of an app. They define behavior and ingest life into the skeleton and skin.
Javascript functions may be called when there is an event triggered from the HTML document. A typical javascript may be called as follows:
In HTML: <span onclick="digit('9')">9</span>
In javascript: function digit(number) { console.log("You entered digit", number); }
which will print the message - You entered digit 9 on the console when 9 is clicked in the HTML.

Javascript can also be used to inject code into a HTML element identified by id. In HTML: <span id="xyz"> </span>
In javascript: document.getElementById('xyz').innerHTML = "Replace span content with this text";
which will display the content Replace span content with this text in the HTML when the span statement happens.

Javascript can also be used to inject code into a HTML element identified by class. In HTML: <span class="abc"> </span>
In javascript: document.getElementByClass('abc')[0].innerHTML = "Replace first span content with this text";
which will display the content Replace first span content with this text in the HTML when the first span statement is encountered.

git

For the class, we will use a client called git for our github repository.

Frequently used git commands include:
  • git clone <url> <directory>
    This command copies content from github repository (identified by <url>) to the directory (identified by <directory>). Please execute the following commands once after cloning:
    • cd <directory>
    • git config --global user.email <youremail@domain.com>
    • git config --global user.name "<YourFirstName YourLastName>"
    • git config http.sslVerify false
  • git add
    This command is needed before git commit if you have created a new directory locally, for example.
  • git commit -am ""
    This command checks in all files that are changed locally. <comment> is used to specify what changes you have made.
  • git push
    This command pushes all commited files to the repository (github). Only after a push, will others be able to see your changes.
  • git pull
    This command pulls the latest frils from the repository (github). You usually do a git clone and frequently do a git pull to pull in changes made by others on your team.
